These are some common types of DPA-style attacks, referenced from Univariate Side Channel Attacks and Leakage Modeling.

1. Single-bit DPA (SB-DPA)

  • Description: Targets only one bit of the sensitive intermediate value during the encryption.
  • Method: Measures leakage (e.g., power consumption) and splits the set of traces into two groups depending on one bit-value (0 or 1).
  • Example:
    Suppose the target cryptographic operation is AES’s S-box output (8-bit value). A SB-DPA attack uses only the first bit of the S-box output. It divides all measurements into two groups:
    • First group: traces where the first bit is 0,
    • Second group: traces where the first bit is 1.
      The attacker computes the difference between the average leakages of these two groups. The largest difference (peak) typically reveals the correct key.

2. All-Or-Nothing DPA (AON-DPA)

  • Description: A specialized form of DPA that uses two extreme values of the leakage function.
  • Method: Compute leakage using a model function (commonly Hamming Weight). Divide traces into two groups based on extreme model values and calculate difference of mean leakages.
  • Example:
    Consider AES S-box output again—its Hamming weight ranges from 0 to 8. AON-DPA uses only Hamming weight values of 0 (all bits zero) and 8 (all bits one). Two groups become:
    • First group: all traces corresponding to S-box output having Hamming weight = 0 (value: 0x00)
    • Second group: traces for outputs with Hamming weight = 8 (value: 0xFF).
      It then measures the difference in leakage averages of these two groups to guess the right key.

3. Generalized DPA (G-DPA)

  • Description: Extends single-bit DPA and All-Or-Nothing DPA by creating more relaxed partition sets.
  • Method: Leakage model (commonly Hamming weight) is split into two general subsets rather than single/extreme values.
  • Example:
    Using AES S-box again, attacker splits leakage into:
    • Group 1: all outputs having Hamming weights from 0 to 4,
    • Group 2: all outputs having Hamming weights from 5 to 8.
      The attacker computes and compares average differences between these two groups. More flexible than SB-DPA or AON-DPA, exploiting greater leakage information.

4. Partitioning Power Analysis (PPA)

  • Description: Extension that splits the leakage dataset into multiple partitions and uses weighted means for distinguishing.
  • Method: Leakage data is divided into several partitions, each associated with carefully chosen weights, and the weighted sum of averages is computed.
  • Example:
    Suppose the AES S-box again has leakage partitioned by exact Hamming weight. The attacker chooses different weights for each Hamming weight class:
    • HW=0 weight -4, HW=1 weight -3, HW=2 weight -2, HW=3 weight -1,
      HW=4 weight 0, HW=5 weight 1, …, HW=8 weight 4.
      The final distinguisher is the sum of each weight multiplied by the corresponding average leakage. It generalizes the notion of grouping and weighting extensively.

5. Correlation Power Analysis (CPA)

  • Description: Uses Pearson’s correlation coefficient as a statistical measure of linear dependence between a leakage model hypothesis and observed leakages.
  • Method: Compute the correlation coefficient to reveal the correct key hypothesis that maximizes the correlation.
  • Example:
    An attacker assumes that the power leakage model on AES S-box output is linearly related to its Hamming weight. For each possible key, she computes the correlation coefficient between the predicted Hamming weight vector and the actual leakage measurements. The correct key is the one whose hypothetical leakage is most correlated to actual measurements.

6. Multiple-DPA (M-DPA)

  • Description: Combines multiple Single-Bit DPAs to exploit information from all bits at once.
  • Method: Perform SB-DPA separately on each bit coordinate, then sum the results.
  • Example:
    For AES S-box (8 bits), attacker computes SB-DPA individually eight times (one per each bit) and then sums all absolute distinguishers (or raw distinguishers). The sum helps increasing efficiency by aggregating more leakage information from multiple bits rather than just one.

7. Absolute Sum DPA (AS-DPA)

  • Description: Similar to multiple-DPA but uses absolute values to achieve robustness to the sign or direction of leakage.
  • Method: Compute SB-DPA separately for each bit, take absolute values, then sum them.
  • Example:
    Again, the AES S-box: attacker computes a SB-DPA for every bit separately, takes absolute value of each bit’s DPA outcome, and then adds them. This avoids cancellation effects that can occur in basic Multiple-DPA if some bits leak positively and others negatively.

8. Linear Regression Attack (Robust Side Channel Attack)

  • Description: Does not rely on choosing a leakage model in advance, rather it builds the leakage model directly from measurement data using regression analysis techniques.
  • Method: The attacker applies linear (or polynomial) regression over the observed data to automatically recover a model that best fits the leakages for each key hypothesis.
  • Example:
    Suppose the device leaks power linearly but with unknown coefficients related to each bit of AES S-box output (for instance, leakage ~ 0.3 × bit0 + 0.6 × bit1 - 0.1 × bit2 + …). An attacker applies regression analysis with traces to automatically determine these unknown coefficients directly from data. The attacker identifies correct key hypothesis by selecting the one whose corresponding regression-fit-model matches actual recorded leakages most closely.
